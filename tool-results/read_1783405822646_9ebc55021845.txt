     1→# CARTINT v2 — Worklog

     3→## Project Overview
     4→CARTINT (Automotive Threat Intelligence) dashboard rebuilt as a Next.js 16 app.
     5→Replaces the original single-source (ransomware.live) static dashboard with a
     6→multi-source dark-web OSINT platform that LLM-classifies every scraped item to
     7→eliminate false positives, and maps accepted threats to the Auto-ISAC ATM.
     8→
     9→## Architecture
    10→- **Framework**: Next.js 16 (App Router) + TypeScript + Tailwind 4 + shadcn/ui
    11→- **DB**: Prisma + SQLite (`Threat`, `Source`, `ScrapeLog`, `Report`)
    12→- **AI**: z-ai-web-dev-sdk — `page_reader` for dark-web/OSINT fetching,
    13→  `chat.completions` for the automotive-relevance classifier (false-positive gate)
    14→  and CTI report generation.
    15→- **Charts**: recharts. **Animation**: framer-motion.
    16→
    17→### Sources (ransomware.live is now only ONE of six — the user's core complaint)
    18→1. `ransomware.live` (ransomware-api) — leak-site victims, pre-filtered + LLM-filtered
    19→2. `ahmia-darkweb` (darkweb-search) — REAL Tor hidden-service search via Ahmia clearnet gateway
    20→3. `darkforum-intel` (darkforum-intel) — analyst-curated Tor forum/marketplace monitoring
    21→4. `bleepingcomputer` (security-rss) — dark-web breach reporting, automotive-filtered
    22→5. `thehackernews` (security-rss) — threat-actor activity, automotive-filtered
    23→6. `nvd-cve` (cve) — NIST NVD automotive / vehicle / CAN-bus / ECU CVEs
    24→
    25→### False-positive prevention (zero false positives)
    26→- `src/lib/scraper/classifier.ts` — every scraped `RawItem` is batch-classified by
    27→  the LLM: `isAutomotive`, `relevanceScore` (0-100), `automotiveCategory`,
    28→  `atmTactic`, `atmTechnique`, `severity`, `classificationReason`.
    29→- A deterministic `quickReject` pre-filter drops obvious non-automotive sectors
    30→  (hospitals, schools, municipalities) before spending an LLM call.
    31→- Only `isAutomotive=true && relevanceScore >= 70` surface as real threats.
    32→  Rejected items are still stored for the audit panel.
    33→- `ScrapeLog` records `fetched / accepted / rejected` per run → false-positive
    34→  rejection rate KPI.
